2.4.1. 单行注释

·[强制][rule038] 必须独占一行。// 后跟一个空格，缩进与下一行被注释说明的代码一致。

2.4.2. 多行注释

·[建议] 避免使用 /*...*/ 这样的多行注释。有多行注释内容时，使用多个单行注释。

2.4.3. 文档化注释

·[强制] 为了便于代码阅读和自文档化，以下内容必须包含以/**...*/ 形式的块注释中。

解释
  1. 文件
  2. namespace
  3. 类
  4. 函数或方法
  5. 类属性
  6. 事件
  7. 全局变量
  8. 常量
  9. AMD 模块

·[强制] 文档注释前必须空一行。

·[建议] 自文档化的文档说明 what，而不是 how。

2.4.4. 类型定义

·[强制] 类型定义都是以 { 开始, 以 } 结束。
